How to earn with the network?


More stable, less gains
You can buy a domain (page name) and earn a small amount for each access to it. That means, if you buy the domain 'car,' for example, every time someone types or searches the word 'car,' you will earn a small sum. 		 

More volatile and bold
You can buy a slot to become an node and earn according to the files you host and others validations.

What is the best for you?


The goal of our project is to create a decentralized search system that can be run on any device. In other words, you'll be able to search and store websites using your own computer, without the need for intermediaries or large search engines. Optionally, you can participate in our network to earn rewards based on the number of sites you find or store, sharing the results with other sites or nodes. Our project uses various patterns to facilitate the download and sharing of files.

hashes - In this directory, each file is saved with a sha-256 name followed by the original file extension (for example, 'aa2239c17609b21eba034c564af878f3eec8ce83ed0f2768597d2bc2fd4e4da5.txt'). By default, text files (with the '.txt' extension) will contain only an HTTP address or URL (for example, http://google.com).

files - In this directory, each file will be saved in an ordered manner, for example, '1.txt, 2.txt, 3.txt,' and so on, making it easy to navigate between them or download all the files of the same type.

contents - Here there will be only HTML files. Typically a text filename, for example, 'cars.html,' and a list of the most relevant websites on the 'cars' theme. Dates use the day-month-year format without delimiters and without leading zeros in single-digit numbers, for example, there might be a page 'cars_1102023' created on the first of October 2023.

The HTML files also can have a sha-256 filename with the information of a respective file.

servers - Files with the addresses of active servers.

audit - files containing addresses that checked or discovered others, for example, 'http://example.com, http://example2.com,' meaning 'http://example.com' found or added 'http://example2.com'.

Or 'http://example.com,aa2239c17609b21eba034c564af878f3eec8ce83ed0f2768597d2bc2fd4e4da5.txt' meaning that 'http://example.com' has the file 'aa2239c17609b21eba034c564af878f3eec8ce83ed0f2768597d2bc2fd4e4da5.txt'.

mastering - Store the addresses of servers storing all available files.


The default hash for file names is sha-256 along with the original file extension.

We want an internet where it's very easy for a website to copy and share content from others, automatically paying a fair fee to them.

We want to raise funds and financing for the creation of an innovative project that will offer a service similar to Google search. However, instead of using a centralized server to store and conduct searches, any computer can participate in the network by both searching for new domains and storing website content. This model, using open-source code and community support, has several advantages, such as making its cost cheaper and offering rewards to users who use its service. The nodes can communicate with each other by selling and sharing data.


Can you imagine how convenient it would be to download automatically the best videos or files from your favorite category while also earning a margin for hosting them?



Unlike traditional search engines where the relevance of a web address corresponds to the number of pages that link to it, our ranking is primarily based on the number of nodes storing the same file or address. Therefore, servers or nodes with more hosted files gain more and have greater relevance.

Our system aims to follow an architecture of several interrelated small applications. This offers great flexibility for users to choose what they want to use from our system and what they want to offer. Additionally, we want to create various types of nodes, each performing a specific task, so that a wide range of devices can run our application, from old computers to even unconventional equipment.

It's not just an open-source platform. It's a platform where each component operates autonomously, and you have maximum flexibility to choose what you want to use and what you want to offer.
